Pengenalan Web Security

Pahami cara kerja komunikasi web, kelemahannya, dan bagaimana menjaga sistem tetap aman dari berbagai serangan umum.

> Apa Itu Web Security?

Web Security adalah praktik untuk melindungi aplikasi web dari ancaman seperti pencurian data, akses ilegal, dan eksploitasi celah. Tujuannya adalah menjaga kerahasiaan, integritas, dan ketersediaan layanan web.

> HTTP vs HTTPS

HTTP tidak dienkripsi, membuat data rentan disadap. HTTPS menggunakan SSL/TLS untuk mengenkripsi komunikasi antara browser dan server.

  • Selalu gunakan HTTPS di semua halaman, terutama login dan data sensitif.
  • Gunakan sertifikat dari CA terpercaya (seperti Let’s Encrypt).

> Cookies, Session & Local Storage

Cookies disimpan di browser dan dikirim otomatis ke server. Session disimpan di server dan hanya ID-nya di browser. LocalStorage disimpan di browser dan tidak otomatis terkirim ke server.

⚠️ Jangan pernah simpan data sensitif di LocalStorage karena mudah dieksploitasi melalui XSS.

> Authentication vs Authorization

Authentication adalah proses memverifikasi identitas user.
Authorization adalah proses menentukan hak akses setelah user terverifikasi.

Pastikan proteksi URL dilakukan di sisi server, bukan hanya UI.

> Ancaman Umum

  • XSS: Serangan dengan menyisipkan JavaScript berbahaya ke halaman web.
  • CSRF: Menyalahgunakan sesi user untuk mengirim aksi palsu ke server.
  • SQL Injection: Manipulasi query SQL melalui input yang tidak divalidasi.
  • Broken Access Control: Pengguna biasa bisa mengakses resource terbatas.

> Security Headers Penting

  • Content-Security-Policy: Mencegah XSS dan penyisipan script eksternal.
  • Strict-Transport-Security: Paksa penggunaan HTTPS.
  • X-Frame-Options: Cegah clickjacking.
  • X-Content-Type-Options: Blok MIME sniffing.
  • Referrer-Policy: Atur informasi yang dibagikan ke referer.

> Tools Penting untuk Testing

  • Burp Suite: Intercept request & eksplorasi celah.
  • OWASP ZAP: Scanner gratis untuk web vuln.
  • Postman: Uji API dan response header.
  • Nikto, sqlmap, nuclei: Untuk scanning otomatisasi.

> Studi Kasus Singkat

XSS di Komentar: Seorang attacker menyisipkan kode <script>...</script> ke kolom komentar. Saat user lain membuka, script berjalan otomatis dan mencuri cookie.

Solusi: escape output, validasi input, dan gunakan CSP.

> Mengenal OWASP Top 10

OWASP (Open Web Application Security Project) merilis daftar 10 besar kelemahan aplikasi web yang paling umum. Ini adalah standar industri dalam menilai keamanan aplikasi.

  • Broken Access Control
  • Cryptographic Failures
  • Injection (termasuk SQLi, NoSQLi)
  • Insecure Design
  • Security Misconfiguration
  • Vulnerable and Outdated Components
  • Identification and Authentication Failures
  • Software and Data Integrity Failures
  • Security Logging and Monitoring Failures
  • Server-Side Request Forgery (SSRF)

> Teknik Mitigasi Tambahan

  • WAF (Web Application Firewall): Saring traffic masuk untuk mencegah serangan otomatis.
  • Rate Limiting: Batasi jumlah request untuk mencegah brute force.
  • CAPTCHA: Lindungi form login dan registrasi dari bot.
  • Audit Log: Catat aktivitas penting seperti login, akses admin, dll.
  • Security Headers: Tambahkan header seperti `CSP`, `HSTS`, dan `X-Frame-Options`.

> Contoh Kasus: SQL Injection

Kode Kerentanan:

"SELECT * FROM users WHERE username = '" + username + "' AND password = '" + password + "'";

Payload: ' OR '1'='1

Hasilnya: Login berhasil tanpa kredensial sah.

Mitigasi: Gunakan parameterized queries atau prepared statements.

> Rekomendasi Implementasi di Dunia Nyata

  • Gunakan framework yang memiliki proteksi built-in seperti Laravel atau Django.
  • Jangan deploy database langsung ke internet.
  • Backup secara berkala dan simpan di lokasi terpisah.
  • Scan otomatis sebelum dan sesudah deploy (CI/CD).
  • Berikan training keamanan pada developer dan QA.
